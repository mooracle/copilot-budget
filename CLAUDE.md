# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Project Overview

**Copilot Budget** is a VS Code extension that tracks GitHub Copilot token usage and estimated cost in AI Credits (AIC; 1 AIC = $0.01) across sessions and optionally appends AI budget git trailers to commit messages. It activates on startup, restores stats from any previous session, polls Copilot session files every 30 seconds, and displays estimated cost in the status bar. Cost is derived from server-reported token counts and a per-model rate card mirrored from `github/docs:data/tables/copilot/models-and-pricing.yml`.

**Two-mode design with auto-detection.** Files mode (default) reads `workspaceStorage/<hash>/chatSessions/*.jsonl` and treats every prompt token as fresh `input` (no heuristic — never undercounts). Editor surfaces (status bar, tooltip, panel) carry a `~` prefix in Files mode to flag the upper-bound estimate. The tracking file's `TR_` lines and the resulting commit trailers carry bare numeric values regardless of mode — decoration is editor-only. Telemetry mode is selected automatically when `github.copilot.chat.otel.dbSpanExporter.enabled = true` AND `<globalStorage>/github.copilot-chat/agent-traces.db` exists; it reads measured `input_tokens` / `output_tokens` / `cached_tokens` per span and drops the tilde. The extension only ever writes `true` to the upstream OTel setting (strictly asymmetric) — disabling Telemetry mode requires the user to flip the upstream setting via VS Code Settings.

## Build & Development Commands

```bash
npm run compile          # Build: src/extension.ts → dist/extension.js (esbuild)
npm run watch            # Watch mode with auto-rebuild
npm run lint             # ESLint on src/**/*.ts
npm test                 # Run all Jest tests
npm test -- --watch      # Jest watch mode
npm test -- <pattern>    # Run single test file (e.g. npm test -- tracker)
npm run package          # Create .vsix via @vscode/vsce
npm run update-rates     # Refresh data/models-and-pricing.yml from github/docs upstream
```

The rate card (`data/models-and-pricing.yml`) is a byte-identical mirror of `github/docs:data/tables/copilot/models-and-pricing.yml`. When GitHub publishes new pricing, run `npm run update-rates` and commit the updated YAML alongside the diff so reviewers see the change directly.

Debug: Press F5 in VS Code to launch Extension Development Host.

## Architecture

The extension follows a **baseline/delta** model: on activation it snapshots Copilot's current cumulative usage as a baseline, then reports only the delta (usage during the current VS Code session). The snapshot source depends on the mode — JSONL session files in Files mode, OTel spans in Telemetry mode.

**Entry point**: `src/extension.ts` — `activate(context)` branches on `context.storageUri`. When defined (workspace open) it picks a `Source` (Files or Telemetry) via `pickSource(context, sessionIdsFn)`, constructs a `Tracker(source, mode)`, restores previous-session stats from the tracking file (if present), wires up the status bar, registers commands (including `showStats`/Open Panel + `showDiagnostics`), optionally auto-installs the git hook, and subscribes to `onDidChangeConfiguration` for hot-swapping the source when the upstream OTel setting flips. A one-time info message ("Switched to Telemetry mode …") fires on the first Files→Telemetry auto-swap per window (gated by `context.workspaceState['copilot-budget.modeSwapMessageShown']`). When `storageUri` is undefined (empty window) it skips tracker/tracking-file/hook setup entirely, renders a static `$(circle-slash) Copilot Budget` status bar bound to `showDiagnostics`, and registers the other commands as info-message handlers.

**Core modules** (all in `src/`):

- **tracker.ts** — Central state machine and Source strategy host. `Tracker` is constructed with a `Source` instance + `mode: 'files' | 'telemetry'`; the mode is threaded into `TrackingStats.mode` for downstream consumers (formatter, trailer writer). Polls every 30s, emits `onStatsChanged` events, exposes `swapSource(newSource, newMode)` for hot-swap on config change (preserves `previousStats` so cumulative totals survive). Returns `TrackingStats` with per-model `{inputTokens, outputTokens, cacheReadTokens, cacheCreationTokens, costAic}` plus totals `{totalTokens, interactions, totalAiCredits, mode}`. The `Source` interface (`{ scan(): Promise<RawAggregateBatch>; dispose(): void; }`) has two implementations:
  - **`JsonlSource`** — preserves the original JSONL polling logic. Caches parsed results by file mtime in `FileCache` entries (each carrying `lastOffset` — a JS string code-unit index after the last `\n` parsed, never bytes — and a `parserState | null` for incremental delta parsing via `applyDeltaLines` + `aggregateFromState`). Falls back to full re-parse when `parserState` is null (evicted/first scan), on truncation, or same-size in-place rewrite. A `MAX_PARSER_STATES = 3` LRU caps retained parser states; evicted entries keep their aggregate and `lastOffset` but lose incremental capability until the next mtime change re-installs `parserState`.
  - **`OTelSource`** — reads measured spans from `agent-traces.db` via `OTelReader.readSpansSince(baseline, sessionIds)`. Baseline = `getLatestTimestamp()` at construction. Session-id filter is best-effort window scoping; same-repo dual-window remains last-writer-wins on the tracking file (pre-existing limitation).
  
  Also exports `RestoredStats` type. `setPreviousStats(restored)` merges a prior session's stats into the current delta, enabling persistence across restarts.
- **sessionDiscovery.ts** — Discovers Copilot session files for the current window only via `discoverSessionFiles(storageUri)`. Resolves `<workspaceStorage>/<hash>/chatSessions/` from `vscode.Uri.joinPath(storageUri, '..', 'chatSessions')` (one `..` since `storageUri` already points at the extension's subfolder), then scans for `.jsonl` files using `NON_SESSION_PATTERNS` and a zero-byte skip. Legacy plain-`.json` chat sessions (from older Copilot Chat versions) are intentionally excluded since the parser is delta-only. When `storageUri` is undefined returns `[]` (empty window). Exports `getDiscoveryDiagnostics(storageUri)` returning `{platform, homedir, storageUri, chatSessionsDir, filesFound}` — `storageUri`/`chatSessionsDir` are `null` in the empty-window state. No cross-variant scanning, no `emptyWindowChatSessions` fallback — empty-window chats are intentionally not billed.
- **sessionParser.ts** — Parses delta-based JSONL chat session files. Pure passthrough: reads server-reported `result.metadata.{promptTokens, outputTokens, cacheReadTokens?, cacheCreationTokens?}` per request and honors what's present, defaulting missing fields to `0`. No turn-based heuristic — when `cacheReadTokens` is absent, it stays `0`. `inputTokens` is derived as `promptTokens - cacheRead - cacheCreation`. Normalizes per-request `modelId` via `getRateCard` so aggregation keys match the rate card. Also returns per-model interaction counts (`modelInteractions`) for diagnostics. Exposes a stateful API for incremental parsing: `ParserState`, `createParserState()`, `applyDeltaLines(lines, state)` (mutates `state.sessionState` via the existing `applyDelta` loop; invalid JSON lines skipped), and `aggregateFromState(state)` (runs `processRequests` over `state.sessionState.requests`). The one-shot `parseSessionFileContent(content)` is a thin shim — `applyDeltaLines(lines, createParserState())` then `aggregateFromState` — so existing callers and fixtures are byte-identical.
- **otelReader.ts** — Opens `<copilot-chat globalStorage>/agent-traces.db` readonly via `node:sqlite` (`new DatabaseSync(path, { readOnly: true })`). Exports `SpanRow`, `OTelReader` interface (`isAvailable()`, `readSpansSince(sinceMs, sessionIds): SpanRow[]`, `getLatestTimestamp(): number`, `close()`), `createOTelReader(globalStorageUri)`, `resolveOTelDbUri(globalStorageUri)`, and `diagnoseUnavailable(globalStorageUri, upstreamSettingEnabled)`. SQL filter is `WHERE operation_name = 'chat' AND start_time_ms >= ?` plus an optional `IN` clause for session ids. Joins `span_attributes` to recover `gen_ai.usage.cache_creation.input_tokens` per span (LEFT JOIN — older spans may not have it). Schema reference (`spans` denormalized columns + `span_attributes`) is documented at the top of the file. The DB handle is owned by `OTelSource`; `OTelSource.dispose()` calls `reader.close()`, and `Tracker.dispose()` (invoked from `deactivate()`) propagates dispose to the active source, so the handle is released on extension deactivation.
- **tokenRates.ts** — Loads and normalizes the per-model rate card from `dist/models-and-pricing.json` at first use (lazy `fs.readFileSync` + `JSON.parse`). The JSON is generated at build time from `data/models-and-pricing.yml` (byte-identical mirror of upstream) by `esbuild.js`, so the runtime bundle has no `js-yaml` dependency. At load time each rate field is multiplied by 100 so stored rates are AIC per 1M tokens (the source YAML on disk stays USD). Exports `RateCard` type, `getRateCard(modelId)` (strips `copilot/`, `copilotcli/`, `claude-code/` prefixes, then exact match against normalized ids — no family fallback), `computeCost(modelId, tokens)` returning AIC per `(input × rate.input + cacheRead × rate.cachedInput + cacheCreation × (rate.cacheCreation ?? rate.input) + output × rate.output) / 1e6`, `getDisplayName(modelId)`, and `getAllRates()`. Unknown models return `null`/`0` to avoid silent mispricing.
- **amountFormatter.ts** — Pure formatting primitive used by status bar, tooltip, panel, and trailer writer. Exports `formatAmount(amountAic, { mode, currency, precision })`: tilde prefix when `mode === 'files'` (dropped for zero); AIC short = `Math.ceil(amount) + ' AIC'`; AIC full = `amount.toFixed(2) + ' AIC'`; USD short = `'$' + (Math.ceil(amountAic) / 100).toFixed(2)`; USD full = `'$' + (amountAic / 100).toFixed(2)`. Currency comes from `getDisplayCurrency()`; mode comes from `TrackingStats.mode`. Used by editor surfaces only — the trailer writer in `trackingFile.ts` formats numbers directly without this helper.
- **statusBar.ts** — Status bar item showing total estimated cost via `formatAmount(..., precision: 'short')`. Tooltip shows total + per-model rows via `formatAmount(..., precision: 'full')` plus a mode-aware disclosure: Files mode reads *"Estimate assumes no caching (upper bound)."*; Telemetry mode reads *"Measured via Copilot's OTel database."*. No standalone quick pick — clicking the status bar opens `showBudgetPanel` (see `budgetPanel.ts`).
- **budgetPanel.ts** — `showBudgetPanel({ tracker })` renders the Copilot Budget panel as a VS Code QuickPick. Three toggle rows at the top (OTel, currency, commit hook), separator, total + per-model breakdown, separator, Refresh. The OTel row is **strictly asymmetric**: when upstream is `false`, accepting flips it to `true` via `configuration.update(..., ConfigurationTarget.Global)` and prompts to reload; when upstream is `true`, accepting opens VS Code Settings (the panel never writes `false`). Mode/currency/upstream-state are re-read fresh on every render via `tracker.getStats()`, `getDisplayCurrency()`, `isOTelDbExporterEnabled()` — no `PanelContext` snapshot to go stale across re-renders.
- **trackingFile.ts** — Reads and writes stats to `<gitdir>/copilot-budget` in key=value format for the commit hook to read. `writeTrackingFile()` is async, takes `stats: TrackingStats` (with `mode`), and uses `resolveGitDir` (with `vscode.Uri`) and `writeTextFile` from `fsUtils`. Each worktree gets its own tracking file. Schema: `SINCE`, `MODE` (`files|telemetry`), `INTERACTIONS`, `TOTAL_AI_CREDITS`, per-model `MODEL_<sanitized>_{INPUT,OUTPUT,CACHE_READ,CACHE_CREATION}_TOKENS` plus `_COST_AIC`, and `TR_`-prefixed trailer lines. The `TR_` lines are controlled by `getTrailerConfig()` from `config.ts`. All `TR_` trailer values are bare two-decimal numbers (no `~`, no `$`) — the trailer name conveys the unit. The `_COST_AIC` machine-readable keys stay unprefixed as before. Per-model AI Credits trailer uses `getDisplayName()` from `tokenRates.ts`. When the opt-in `estimatedCost` trailer is enabled, the USD value is derived inline from `totalAiCredits / 100` at trailer-write time. `parseTrackingFileContent(content)` requires at least `SINCE` and `TOTAL_AI_CREDITS`; legacy v0.6.x files containing `TOTAL_COST_USD` / `_COST_USD` keys are tolerated by silently ignoring those keys (same path now ignores any future unknown key, including `MODE` for forward-compat with older readers). Pre-0.6 files (no `TOTAL_AI_CREDITS`) return `null`.
- **fsUtils.ts** — Thin wrappers around `vscode.workspace.fs` to avoid repetitive TextEncoder/TextDecoder boilerplate. Exports `readTextFile(uri)`, `writeTextFile(uri, content)`, and `stat(uri)`. All return promises and handle errors gracefully (returning `null` on failure). Used by `gitDir.ts`, `trackingFile.ts`, and `commitHook.ts`.
- **gitDir.ts** — Shared utility for resolving git directories. `resolveGitDir(workspaceRoot: vscode.Uri): Promise<vscode.Uri | null>` follows `.git` files (worktrees, submodules, devcontainers) to the real git dir. `resolveGitCommonDir(workspaceRoot: vscode.Uri): Promise<vscode.Uri | null>` further follows the `commondir` file in worktrees to reach the shared git dir (where hooks live). Uses `fsUtils` helpers for filesystem access via `vscode.workspace.fs`.
- **commitHook.ts** — Installs/uninstalls a POSIX `prepare-commit-msg` hook that reads `copilot-budget` from the git dir and appends git trailers to commit messages. The hook uses a generic `TR_` line protocol: if the tracking file contains any `TR_<name>=<value>` lines, the hook converts them to git trailers via sed (e.g., `TR_Copilot-AI-Credits=42.31` becomes `Copilot-AI-Credits: 42.31` (mode-independent)); otherwise it skips. Trailer names are configured in `config.ts` and written by `trackingFile.ts`. All exported functions (`isHookInstalled`, `installHook`, `uninstallHook`) are async. Uses `resolveGitCommonDir` (with `vscode.Uri`) and `readTextFile`/`writeTextFile` from `fsUtils`. Includes a `makeExecutable` helper that uses `fs.chmodSync` for local files and a VS Code shell task for remote filesystems. The hook script uses `git rev-parse --git-dir` at runtime for worktree-safe tracking file resolution. Resets the tracking file after appending (non-squash path only). On `$2 == squash` (also fires for `fixup -c` / `fixup -C`) the hook sums duplicate trailer lines whose value is a bare number — in practice this is the default `Copilot-AI-Credits:` total. The opt-in aggregate per-model trailer (`Copilot-AI-Credits-Models: A=N,B=M`) has a non-numeric value and is not matched; the `Copilot-Est-Cost` trailer and any user-renamed `aiCredits` key are also not matched (the awk regex hard-codes `Copilot-AI-Credits`). The squash path does NOT consult the tracking file or reset it. Plain `fixup` never triggers the hook (git discards the message).
- **config.ts** — Wraps `copilot-budget.*` settings plus the upstream `github.copilot.chat.otel.dbSpanExporter.enabled` read path. Exports `TrailerConfig` interface and `getTrailerConfig()`, `getDisplayCurrency(): 'aic' | 'usd'`, `isOTelDbExporterEnabled(): boolean`, `getEstimationMode(reader, upstreamEnabled?)` (returns `'telemetry'` iff `upstreamEnabled` AND `reader.isAvailable()`; otherwise `'files'`), and `onDidChangeOTelSetting(cb)`. Defaults: `aiCredits` → `"Copilot-AI-Credits"` (on; primary trailer), `estimatedCost` → `false` (opt-in USD trailer), `aiCreditsPerModel` → `false` (opt-in), `displayCurrency` → `"aic"`.
- **utils.ts** — Shared utility helpers. Exports `sanitizeModelName(name)` (replaces non-alphanumeric characters with underscores for key=value tracking file keys) and `errorMessage(err)` (safely extracts a message string from unknown thrown values). Used by `tracker.ts`, `trackingFile.ts`, and `commitHook.ts`.
- **logger.ts** — Shared OutputChannel logger singleton. Exports `log()` (timestamped append), `getOutputChannel()`, and `disposeLogger()`. Used by sessionDiscovery, tracker, and the diagnostics command.

## Testing

Tests live alongside source files as `*.test.ts`. The `vscode` module is mocked via `src/__mocks__/vscode.ts` (includes `workspace.fs`, `Uri.joinPath`, `FileType` enum, `Task`/`ShellExecution` mocks, `ConfigurationTarget` enum, `QuickPickItemKind`, and a `__workspaceUpdate` `jest.fn` capturing every `getConfiguration(section).update(key, value, target)` call for asymmetric-write assertions). Tests for workspace-side modules (`gitDir`, `trackingFile`, `commitHook`) mock `./fsUtils` rather than Node.js `fs`. Session discovery and parser tests still mock `fs` directly. `otelReader.test.ts` synthesizes a fixture SQLite DB in `beforeAll` via real `node:sqlite` — no checked-in binary. Tests heavily use `sessionDiscovery`, `sessionParser`, and `otelReader` mocks to isolate units.

## Key Design Details

- **Two-mode auto-detection** — `getEstimationMode()` returns `'telemetry'` iff `github.copilot.chat.otel.dbSpanExporter.enabled` is `true` AND `OTelReader.isAvailable()` returns `true`. Otherwise `'files'`. On `onDidChangeConfiguration` for the upstream key, `extension.ts` disposes the old `Source`, picks a fresh mode, constructs the new `Source`, and calls `Tracker.swapSource(newSource, newMode)` — `previousStats` carries over so cumulative totals survive the swap.
- **Asymmetric upstream write** — the extension only ever calls `configuration.update('github.copilot.chat.otel', 'dbSpanExporter.enabled', true, ConfigurationTarget.Global)`. To disable Telemetry mode, the user must flip the upstream setting via VS Code Settings. The panel's OTel row, when upstream is already `true`, deep-links to `workbench.action.openSettings` filtered by the setting query string. There is an explicit test asserting that accepting the OTel row when upstream is `true` does NOT call `configuration.update`.
- **Window-scoped discovery** — each VS Code window only scans its own `workspaceStorage/<hash>/chatSessions/` (Files mode) or filters OTel spans by the current window's session-id set (Telemetry mode). Same-repo dual-window remains last-writer-wins on the tracking file (shared hash → shared chatSessions dir → shared tracking file path); pre-existing limitation, out of scope.
- **Remote-host limitation** — `agent-traces.db` lives where Copilot Chat runs (typically workspace-side in remote dev). When the upstream setting is `true` but the DB is missing locally, `diagnoseUnavailable()` returns a one-line message; Tracker logs it once at activation and silently falls back to Files mode. `extensionKind` stays `["ui","workspace"]` to preserve empty-window activation.
- **No runtime YAML dependency** — `js-yaml` is dev-only. The build step in `esbuild.js` parses `data/models-and-pricing.yml` and writes `dist/models-and-pricing.json`; the runtime loader (`tokenRates.ts`) uses `JSON.parse`. Tests generate `src/__fixtures__/models-and-pricing.json` from the YAML fixture via `jest.globalSetup.js`.
- **Workspace-side file I/O uses `vscode.workspace.fs`** — Modules that access workspace files (`gitDir.ts`, `trackingFile.ts`, `commitHook.ts`) use `vscode.workspace.fs` via `fsUtils.ts` wrappers, enabling transparent remote filesystem proxying in devcontainers, Codespaces, and SSH Remote. Only `commitHook.ts` retains a Node.js `fs` import for local `chmodSync`. The host-side modules `sessionDiscovery.ts` and `otelReader.ts` use Node.js `fs` / `node:sqlite` since they read from the host machine's filesystem.
- **esbuild bundles** to a single `dist/extension.js` (CommonJS, **Node 22 target** — required for `node:sqlite`, vscode external). The build also converts `data/models-and-pricing.yml` to `dist/models-and-pricing.json`. `engines.vscode` is pinned to `^1.103.0` (first release bundling Electron 37 / Node 22.17.0, where `node:sqlite` is stable); pre-1.103 VS Code installs cannot load the extension.
- The commit hook is pure POSIX shell with no external dependencies.
- Prototype pollution prevention is implemented in the JSONL delta parser.
- The extension makes no network calls at runtime. All session data is read from local files (JSONL or SQLite); the rate card is shipped in the extension bundle and refreshed by contributors via `npm run update-rates`. Only aggregate token counts are read from OTel spans — prompt and response content are never touched.
